Generate tls certificate

1.Install the acem.sh certificate generation tool. The following 3 methods are provided. Choose any one of the methods to install the certificate tool (Reminder: automatically upgrade acme.sh and enter acme.sh upgrade under root)

2. Generate the certificate path under the /root/.acme.sh file, you can check it yourself after installation
Reminder: There are many ways to generate a certificate through acme.sh:
For example — automatic DNS API integration such as: cloudflare DNS API token and use of global API key acme.sh supports most dns to generate certificates
For example — use DNS manual mode, and many other installation methods, if you are a good learner, you can Google

The generated certificate is as follows: This video only uses the designated port to generate the certificate. **It is recommended to use the 443 port to generate the certificate** (usually a single domain name is sufficient, after all, it is used to overcome the wall and do not need to do so many fancy multi-domains, such as: main domain baidu.com is not recommended to use www.baidu.com, because it is the front-end web that overturns the wall, please customize such as tw.baidu.com, the prefix tw can be customized, please control three characters, the main domain baidu.com and the secondary www .baidu.com can spare you understand)

Apply for a certificate by listening on port 80, if port 80 is occupied, please use port 443, please make sure these ports are open

Listen to port 443 to issue certificates, please make sure port 443 is open

If you use non-standard port 443 behind the reverse proxy or load balancer, you can use –tlsport to specify the port

-k means the length of the key, the following values ​​can be ec-256, ec-384, 2048, 3072, 4096, 8192, with ec means that the generated ECC certificate is, if not, it is the RSA certificate. In terms of security, a 256-bit ECC certificate is equivalent to a 3072-bit RSA certificate

Check whether ssl is enabled. Open the website https://myssl.com/ssl.html and enter your own domain name and port 443 or open https://www.ssllabs.com/ssltest/index.html to query, you can also search for the keyword ssl on Google Query, many websites can query, whether the score has reached A level

Reminder: To generate a certificate through the acme.sh tool, please use the 443 port code as follows: sudo ~/.acme.sh/acme.sh –issue -d domain name –alpn -k ec-256 (listen to port 443 To issue a certificate, please make sure that port 443 is turned on. Follow the procedure of the article to operate port 80. When generating a certificate, you will be prompted that the port is occupied. Of course, if you want to be simpler, you can pass the certificate you have purchased or apply for the pagoda or other certificates, and put the key and certificate directly Copy the copy to the specified path of the server, and configure nginx to point to the certificate and path. Some free certificates will lack the CA certificate, so the website ssl score will be greatly reduced. The solution is to re-add the CA certificate.
